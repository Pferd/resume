Cuedesk Remote Assist

A peer-to-peer product that provides hassle free remote desktop control

Cuedesk Remote Assist is a peer to peer remote desktop control software using WebRTC Framework. It uses networking and Audio Video stack from WebRTC for screencasting and remote control injections. The project programming is highly event driven at the very top level with strong system programming influence. Though it is said P2P, 8~10% of the communication is relayed with STUN/TURN servers which are hosted public. For signalling purpose a XMPP server called Openfire is hosted on public with custom plugin implementations

The user interface is completely browser based with a light weight integrated http server. This leverage a stage for integrating into a web based central administration portal for both the communicating parties.

The web administration portals use Google App Engine platform and Google App Script API's for business logics. The browser user interface uses Bootstrap 3.2 and Javascript

My current role involves end-to-end product development with blended technologies including open source.

QoEMoS - Quality of Experience and Mean Opinion Score

The project focuses on establishing a relation between lower layer (Network and Physical) performance metrics towards end user experience (QoE). My role as a Project Assistant deals with setting up a demonstrator for evaluating the H.264 scalable video codec (SVC) QoE over a real base station with WCDMA capabilities (Rhode and Schwarz, CMU 200). The setup involves accessing different layers of scalable videos from a media-server to a client (video player on Linux) via a controlled real base station. The video player at the client is instrumented to report player performance metrics. With the statistics, a mathematical relation is established between physical layer metrics and user perceived experience (QoE) under varying jitter-buffer conditions. To perform batch runs, automation is performed at the client side using Java programming with Swing UI components.
